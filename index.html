Website Metrics Analysis

Background

This website helps make appointments with doctors across various clinics and days. Users were divided into two groups, A and B, each asked to schedule a specific appointment using a different website interface. Our aim is to determine if these interface changes would influence user behaviors when interacting with the interface. We analyzed misclick rate, time on page, and time to first click between the two groups. Results showed a big difference: Group B spent shorter time on the task. This finding may imply version B is more efficient to use and help improve user experience.

Metric and Hypotheses

  1. 1. Misclick Rate

    a. Definition

    The frequency with which users click something else on the page before finding the correct button for the task.

    b. Null Hypothesis

    The rate of users clicking the correct button for the task remains the same.

    c. Alternative Hypothesis

    More users click the correct button for the task.

    d. Prediction

    Fail to reject the null hypothesis. I didn't change the interface of the available appointments part. So user behavior should not be affected when trying to find the corrent appointment.

  2. 2. Time on Page

    a. Definition

    Time spent on the webpage for each user group.

    b. Null Hypothesis

    Users spend the same time on the website.

    c. Alternative Hypothesis

    Users spent less time on the website.

    d. Prediction

    Reject the null hypothesis. I changed background color and placed buttons more center to the screen. Users are expected to spend less time on the page.

  3. 3. Time to First Click

    a. Definition

    Time spent to click one button for the first time.

    b. Null Hypothesis

    Users spend the same time first clicking one button after entering the website.

    c. Alternative Hypothesis

    Users spend less time first clicking on button after entering the website.

    d. Prediction

    Reject the null hypothesis. I changed background color and hover color of making appointment buttons. Users are expected to spend less time finding the buttons and will get feedback when hovering over the buttons.

Results and Analysis

1. Misclick Rate

a. Results

Metric Value
Average (A) 0.2609
Variance (A) 0.2016
Average (B) 0.0714
Variance (B) 0.0714
T-score 1.6088
P-value 0.0583

b. Statistical Test

I choose One-tailed t-test because I want to learn the misclick rate of which group of users is bigger.

c. Conclusion of Statistical Significance

The difference between versions A and B with respect to the metric is not statistically significant.

d. Analysis

Group A’s misclick rate (0.26) is higner than group B’s (0.07). Since users in two groups were actually the same, this may be attributed to their getting more familiar with the website.

e. Decision

I fail to reject the null hypothesis for P-value is more than 0.05.

Summary

Version A shows higher variances across all metrics measured, indicating a wider spread of data points around the mean. This may imply version A’s data are less accurate compared to version B. And version A’s average time on page and average time to first click are both higher, showing version A’s interface may be less efficient for users to interact with.
